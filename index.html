1.3) What is primary key in table. how to use

The PRIMARY KEY constraint uniquely identifies each record in a table.
Primary key must contain UNIQUE values, and can not contain NULL Values.

Syntax: create table tablename
(
column_name   data_type   not null   default value,
column_name   data_type   not null   default value primary key(column_name)
);

Example:  create table students
(
id   int   not null,
name   varchar(50)   not null   default "unnamed",
class   int   not null   default 1,
roll_no   int   not null,
school_name   varchar(100)   not null   default "janakpur school" primary key(id)
);

MySql ORDER BY Keyword or GROUP BY Keyword

ORDER BY Keyword:

ORDER BY keyword is used to sort the result-set in ascending or descending order.

Syntax: select * from students order by fname;
Syntax: select age from students order by age;
Syntax: select age from students order by age desc;

GROUP BY Keyword

The GROUP BY command is used to group the result set(used with aggregate function: COUNT,MAX,MIN,SUM,AVG).
Syntax: select count(id), fname, lname, from students group by lname;
Syntax: select count(id), fname, lname, from students group by lname order by desc;

LIKE Operator

The LIKE operator is used in a WHERE Clause to search for a specified pattern in a column.
There are two wildcards often used in conjunction with the like operator.

  • '%'
    • The percent sign represents zero,one or multiple characters
    Syntax: select * from students where fname like "ra%";
    Syntax: select * from students where fname like "%ab%";
    Syntax: select fname, lname from students where fname like "ra%";
    Syntax: select age from students where age like "%7";
  • '_'
    • The underscore represents a single character
    Syntax: select * from students where lname like "_adav";
    Syntax: select * from students where lname like "_ada_";
    Syntax: select fname, lname from students where fname like "rab_";
    Syntax: select age from students where age like "_7";

MySQL Logical Operator

The logical operators are those that true or false.
They return a true or false values to combine one or more true or false values.
  • AND(&&)
    • The AND operator is used to compare data with more than one condition.
    If all the conditions returns TRUE then only it will display records.
    Syntax: select * from students where lname = "yadav" and age = 28;
  • OR(||)
    • The OR Operator is used to compare data with more than one condition.
    If either of condition is TRUE it will return data.
    Syntax: select * from students where age > 20 or age < 30;
  • BETWEEN
    • The BETWEEN operator is inclusive: begin and end values are included.
    Syntax: select * from students where age between 20 and 30;
  • IN
    • The IN operator is a shorthand for multiple OR conditions.
    Syntax: select * from students where fname in ('rabi','krishna','krish');
  • NOT
    • Syntax: select * from students where age not between 20 and 30;
    Syntax: select * from students where fname not in ('rabi','krishna','krish');

Primary Key & Foreign Key

  • Primary Key
    • The PRIMARY KEY constraint uniquely identifies each record in a table.
    Primary Keys must contain UNIQUE values, and cannot contain NULL values.
    Syntax1: create table customers(customer_id int not null auto_increment primary key, name varchar(50) not null, address varchar(150) not null default "janakpur");
    Syntax2: create table customers(customer_id int not null auto_increment, name varchar(50) not null, address varchar(150) not null default "janakpur" primary key(customer_id));
  • Foreign Key
    • A FOREIGN KEY is a key used to link two tables together.
    A FOREIGN KEY is a field (or collection of fields) in one table that refers to the PRIMARY KEY in another table.
    Syntax: create table orders(order_id int not null auto_increment primary key, order_datetime datetime, amount int, customers_id int foreign key(customers_id) references customers(customer_id) );

All About JOINS CLAUSE

A JOIN clause is used to combine rows from two or more tables, based on related column between them.
There are different types of JOINs
  • (INNER)JOIN
    • Returns records that have matching values in both tables
    Syntax 1 : select * from customers join orders;
    Syntax 1.1 : select customer_id, name, amount, order_datetime from customers join orders;
    Syntax 2 : select * from customers join orders on customers.customer_id = orders.customers_id;
    Syntax 3 : select * from customers inner join orders on customers.customer_id = orders.customers_id;
  • LEFT(OUTER)JOIN
    • Returns all records from the left table, and the matched records from the right table
    Syntax: select * from customers left join orders on customers.customer_id = orders.customers_id;
  • RIGHT(OUTER)JOIN
    • Returns all record from the right table, and the matched records form the left table
    Syntax: select * from customers right join orders on customers.customer_id = orders.customers_id;
